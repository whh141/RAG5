## 可直接替换的正文

### 论文（设计）的主要内容、理论意义和应用价值

#### （一）论文（设计）的主要内容

本课题面向高校教学服务与学籍管理场景，设计并实现一个基于智能体增强检索生成（Agentic RAG）的校园问答系统。与传统“检索后直接生成”的线性 RAG 不同，本系统以“问题识别准确、证据链可追溯、路由逻辑可验证、回答质量可评估”为核心目标，围绕动态路由、混合检索、证据抽取、引用校验和评测闭环展开。当前项目已形成较完整的实现基础：系统以山东大学本科教学服务相关 PDF、DOCX 文件及 FAQ 数据为知识源，构建了覆盖学籍异动、学分管理、毕业论文、奖助政策等场景的本地知识库，并配套构建了 148 条问答评测集、133 条路由标注集和 60 条越界问题测试集，为后续实验验证提供了可量化的数据基础。

1. 校园异构知识库构建与知识预处理机制设计

本模块的目标是为后续路由、检索和生成提供高质量、结构化的知识底座。项目当前知识源并非单一文本，而是由校园规章制度、办事流程文档、通知附件以及 FAQ 标准问答共同组成，具有来源分散、格式异构、表述风格不统一等特点。因此，本课题首先研究面向校园服务场景的知识清洗与统一表示方法。

具体而言，系统将 PDF、DOCX 等文档解析为统一文本对象，并结合 FAQ 数据形成多源知识集合；在切分阶段，兼顾制度条款的语义完整性和检索粒度，避免因切分过细导致上下文丢失，也避免因切分过粗降低召回精度；在索引阶段，为每个知识片段附加 `doc_id`、`title`、`source_file`、`page`、`chunk_id` 等元数据，使后续生成结果能够回溯到具体文档与页码。该模块不仅服务于向量检索，还为答案引用展示、知识库更新和实验分析提供统一的数据基础。

2. 基于 LangGraph 的严格动态路由与任务编排机制研究

本课题的核心创新之一，不是简单地“让模型决定查本地还是查网络”，而是构建一套具有明确约束的动态路由机制。当前系统将用户问题划分为 `simple_fact`、`complex_reasoning`、`time_sensitive` 三类主任务，并仅在问题明显越界时使用 `ood` 作为辅助异常标签。与之对应，系统仅允许三条执行路径：本地检索 `retrieve_local`、联网检索 `retrieve_web` 和拒答 `refuse`，并通过路由契约严格绑定“意图类型”和“执行路径”的一一对应关系。

这一设计具有明确的业务合理性。高校问答系统不同于开放域闲聊系统，其目标不是“尽量回答”，而是“在正确知识边界内回答”。如果本地知识不足时随意切换到互联网搜索，虽然可能提升表面可回答率，但也会引入非官方来源、过期信息或场景错配，导致业务逻辑偏移。因此，本课题明确采用单链路执行原则：单个子问题一旦完成路由判定，即沿唯一链路执行；即使证据不足、结构输出异常或引用校验失败，也不切换到其他路径，而是将问题暴露为待优化样本。该原则使系统的行为边界更清晰，也使后续实验结论更具可解释性。

在工作流层面，本课题基于 LangGraph 将问答过程组织为“问题拆解 - 意图分类 - 计划生成 - 工具执行 - 答案综合 - 反思评估 - 最终输出”的图结构流程。对于单问题输入，系统直接进入动态路由；对于包含多个子问的复合问题，系统先进行有序拆解，再对子问题逐个独立路由并汇总结果，从而兼顾复杂输入的处理能力与执行链路的确定性。

3. 基于混合检索、证据抽取与引用校验的答案生成机制研究

为提升校园知识问答的准确性与可追溯性，本课题不采用“召回文档后直接拼接生成”的粗粒度方案，而是将本地 RAG 主链路细化为“混合召回 - 去重融合 - 重排序 - 证据抽取 - 答案生成 - 引用校验”六个阶段。

在召回阶段，系统结合 FAISS 向量检索与 BM25 关键词检索，分别建模语义相似性和词项匹配能力，以降低单一路径在制度术语、办事名词和简略提问下的召回偏差；在排序阶段，引入重排序模型对候选文档进行精排，并结合词法得分、向量得分和槽位匹配得分形成最终排序；在证据处理阶段，系统不直接把全部文档交给大模型生成，而是先要求模型仅从候选文档中抽取能够支持回答的核心事实，再基于结构化证据完成答案生成。对于复杂推理问题，系统还会显式生成带证据编号的结构化推理链，以保证对比、归纳和条件判断过程具备可审查性。

更重要的是，系统在生成后执行引用校验：回答中的每个关键结论必须绑定合法证据编号，且引用编号必须能够映射回真实证据项；一旦回答未引用证据，或引用了不存在的编号，即判定该结果不合格。该机制将“证据先行”落实为工程约束，避免大模型在校园制度问答中出现无依据补全和事实幻觉。

4. 基于同路由反思与评测闭环的系统验证平台实现

为验证上述方法在真实校园知识场景中的可用性，本课题进一步实现了系统原型及评测闭环。当前项目已具备两类交互入口：一类是面向演示和人工测试的 Gradio 界面，可展示路由结果、检索来源、证据摘要和引用信息；另一类是基于 FastAPI 的服务接口，支持标准问答调用、SSE 流式输出以及知识库文件上传、删除和重建，便于后续系统集成与扩展。

在质量控制方面，系统引入 Reflection 节点对当前回答进行相关性、准确性、完整性和可用性评估，并在分数不足时仅允许在当前路由内进行有限轮次重试，而不允许跨路径改道。这种“同路由反思”机制既保留了智能体的自我修正能力，又避免了为了追求表面成功率而破坏实验边界。

在实验验证方面，本课题已构建三类评测任务：其一是路由评测，用于验证问题分类与执行路径的一致性；其二是问答评测，用于统计回答的语义相似度、路由正确率和引用存在率；其三是越界问题评测，用于检验系统面对超出校园知识域的问题时是否能够稳定拒答。相比只展示个别问答样例，这种闭环式验证更能说明系统是否真正具备工程可用性和研究分析价值。

#### （二）理论意义和应用价值

##### 1. 理论意义

（1）推动 RAG 系统从“静态线性生成”向“受约束的智能体工作流”演进

传统 RAG 多采用“检索 - 生成”的单步链路，虽然结构简单，但难以处理问题类型差异、复合任务拆解和回答质量自检等需求。本课题通过引入 LangGraph 工作流，将问答任务建模为具有节点分工和状态流转的智能体图结构，并进一步加入路由契约、子问题拆解和反思评估等机制，使 RAG 不再只是“检索器 + 大模型”的松散拼接，而成为一个可编排、可约束、可验证的任务系统。这种从线性管线向图式工作流的转变，对于理解 Agentic RAG 的系统设计方法具有一定理论参考意义。

（2）强调“严格链路约束”在垂直领域问答中的必要性

当前许多智能体系统倾向于通过补救、回退和跨路径兜底来提高表面成功率，但在高校制度、教学服务等垂直场景中，这种策略可能掩盖真实问题来源，甚至让系统在非官方信息基础上输出看似合理但实际不可靠的答案。本课题提出的严格链路约束思想，强调同一问题在单次执行中的来源边界不可漂移，将“不可回答”视为系统行为的一部分而非失败例外。该思路有助于丰富垂直领域 RAG 中关于可控性、边界性和可验证性的研究视角。

（3）探索“证据抽取 + 引用校验 + 结构化推理”一体化的可解释生成机制

大模型在复杂制度问答中常见的问题并不是不会说，而是说得过多、说得过满、说得无法验证。本课题以证据抽取为前置步骤，以引用校验为输出门槛，并对复杂问题引入结构化推理链表示，使“模型为何这样回答”能够被映射到明确证据项。这种从文档片段到事实证据、再到推理步骤和最终回答的分层设计，对于研究可解释问答系统具有较强的方法论价值。

（4）形成面向垂直知识问答的实验评测闭环

仅依赖主观体验或零散案例难以判断 Agentic RAG 的有效性。本课题将路由、回答和拒答分为三类独立评测任务，并结合语义相似度、路由准确率、引用存在率和越界拒答率等指标形成验证闭环。这种评测思路能够将“模型看起来能答”转化为“系统在哪些维度可靠、在哪些样本上失效”的可分析结论，对垂直场景问答系统的实验设计具有借鉴意义。

##### 2. 应用价值

（1）为高校教学服务场景提供低成本、可追溯的智能问答方案

高校教学服务涉及学籍、课程、学分、奖助、毕业论文、实践教学等大量重复性咨询，现有人工答疑成本高、响应慢且易受经验差异影响。本课题通过整合学校现有制度文件、流程文档和 FAQ 数据，构建统一知识服务入口，使学生能够以自然语言直接获取办事地点、材料、流程、收费标准和制度依据等信息，并通过引用来源核验回答可靠性。该系统可直接服务于院系教务咨询、学生事务大厅、图书馆咨询等场景，具有明确的现实应用价值。

（2）提升校园智能服务系统的透明度与治理能力

相比仅返回结论的黑盒式聊天系统，本课题实现的系统能够展示路由依据、检索问题、证据摘要和引用来源，在接口层还支持知识库文件管理和重新向量化操作。这意味着系统不仅可以回答问题，还可以被维护、被审查、被迭代。对于高校这类强调制度规范与责任可追溯的组织环境而言，这种透明化设计比单纯追求“回答流畅”更具治理价值。

（3）为其他垂直知识场景提供可复用的技术框架

本课题虽然以校园教学服务为落地对象，但其方法结构具有较强迁移性。严格动态路由、混合检索、证据抽取、引用校验和评测闭环并不依赖校园场景本身，后续可扩展至企业制度问答、政务知识服务、医疗流程咨询等对准确性和可解释性要求较高的领域。因此，本研究不仅有助于解决当前校园知识服务问题，也可为其他垂直领域智能问答系统的构建提供可复用的参考框架。
